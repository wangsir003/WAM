## ✅ 功能测试清单

### 1️⃣ Release 编译测试

**前提条件：**
- 已添加一个 Android 项目
- 项目已配置签名文件

**测试步骤：**
1. 选择项目
2. 找到"开发者工具箱"卡片
3. 点击"一键 Release 打包"按钮（红色，火箭图标）
4. 观察日志控制台的实时输出
5. 等待编译完成
6. **预期结果：**
   - 日志显示 "✓ RELEASE BUILD SUCCESSFUL"
   - Release APK 文件夹自动打开
   - 显示成功消息

---

### 2️⃣ WiFi 连接测试

**前提条件：**
- 手机已通过 USB 连接
- 手机和电脑在同一 WiFi 网络

**测试步骤：**
1. 在命令行执行：`adb devices`（确认 USB 连接正常）
2. 执行：`adb tcpip 5555`
3. 获取手机 IP（设置 → 关于手机 → 状态）
4. 在应用中输入：`192.168.1.xxx:5555`
5. 点击"连接"按钮
6. **预期结果：**
   - 日志显示 "✓ 连接成功"
   - 设备列表自动刷新
   - 新设备显示为 IP 地址

---

### 3️⃣ 批量截屏测试

**前提条件：**
- 至少连接一个设备

**测试步骤：**
1. 在设备管理区域勾选 1-2 个设备
2. 点击"批量截屏"按钮
3. **预期结果：**
   - 日志显示截屏进度
   - 桌面上出现截图文件
   - 文件命名：`项目名_设备ID_时间戳.png`
   - 显示成功消息：`截屏完成！成功 X/X 个设备`

---

### 4️⃣ 批量操作测试

**前提条件：**
- 项目包名已识别
- 至少选中一个设备

**测试步骤：**

**清理缓存：**
1. 勾选设备
2. 点击"清理缓存"
3. 确认对话框
4. **预期结果：** 日志显示 "✓ 设备 xxx 操作成功"

**强杀应用：**
1. 确保应用正在运行
2. 点击"强杀应用"
3. **预期结果：** 应用被强制关闭

**卸载应用：**
1. 点击"卸载应用"（红色按钮）
2. 确认对话框
3. **预期结果：** 应用从设备上被卸载

---

### 5️⃣ Logcat 监控测试

**前提条件：**
- 至少连接一个设备

**测试步骤：**
1. 在"Logcat 实时监控"下拉框选择设备
2. 点击"开启监控"
3. 观察状态变为"监控中"（绿色闪烁点）
4. 在手机上操作应用，触发一些日志
5. 观察日志控制台的实时输出
6. **预期结果：**
   - 日志实时流式输出
   - 包含 "Exception"、"ERROR" 的日志显示为红色
   - 包含 "WARN" 的日志显示为黄色

**停止监控：**
1. 点击"停止监控"（红色按钮）
2. **预期结果：** 
   - 状态变回正常
   - 日志显示 "⚠ Logcat 监控已手动停止"

---

5. **预期结果：** 应用正常启动，所有功能可用

## 🐛 常见问题

### Q: Release 编译失败
**A:** 检查以下内容：
- 项目是否配置了签名文件
- `gradlew.bat` 是否存在
- 是否有网络连接（首次编译需要下载依赖）

### Q: WiFi 连接失败
**A:** 检查：
- 手机是否执行了 `adb tcpip 5555`
- 手机和电脑是否在同一网络
- IP 地址和端口是否正确

### Q: 截屏保存失败
**A:** 确保：
- 设备有截屏权限
- 桌面路径可访问
- 磁盘空间充足

### Q: Logcat 无输出
**A:** 检查：
- 设备是否正确连接
- 应用是否正在运行
- 是否有日志输出（尝试操作应用）
