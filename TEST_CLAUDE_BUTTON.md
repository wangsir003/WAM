# Claude AI 按钮复用功能测试

## 修复内容
修复了 Claude AI 按钮每次点击都创建新窗口的问题。现在会正确复用已有窗口。

## 测试步骤

### 1. 准备工作
- WAM 应用已启动
- 在左侧项目列表中选择一个项目（或添加一个测试项目）

### 2. 第一次点击测试
1. 点击右侧功能区的 "Claude AI" 按钮（紫色渐变按钮）
2. **预期结果**：弹出一个新的命令行窗口（Windows Terminal 或 PowerShell）
3. 观察窗口是否正常启动 Claude

### 3. 第二次点击测试（核心测试）
1. **不要关闭**第一次打开的 Claude 窗口
2. 回到 WAM 应用
3. 再次点击 "Claude AI" 按钮
4. **预期结果**：
   - 不会创建新窗口
   - 已有的 Claude 窗口会被激活并带到前台
   - WAM 应用显示消息："✓ 已激活现有的 Claude 窗口"

### 4. 多次点击测试
1. 重复步骤 3 多次
2. **预期结果**：每次都应该激活已有窗口，而不是创建新窗口

### 5. 关闭窗口后再次点击
1. 关闭 Claude 窗口
2. 再次点击 "Claude AI" 按钮
3. **预期结果**：创建新的 Claude 窗口（因为之前的已关闭）

## 验证方法

### 方法 1：目测窗口数量
- 打开任务管理器，查看 PowerShell 或 Windows Terminal 进程数量
- 多次点击后，进程数量不应增加

### 方法 2：查看日志
- WAM 应用的日志区域应该显示：
  - 第一次：`✓ Claude AI 已启动`
  - 后续：`✓ 已激活现有的 Claude 窗口`

### 方法 3：检查锁文件
打开目录：`.wam-cache/locks/`
- 第一次点击后，应该有一个以项目名命名的 `.lock` 文件
- 关闭 Claude 窗口后，锁文件应该被删除

## 问题排查

如果问题仍然存在：
1. 检查控制台日志（F12 开发者工具）
2. 查看 `.wam-cache/locks/` 目录的锁文件状态
3. 检查是否有残留的 PowerShell 进程

## 技术细节

修复的核心逻辑：
1. 每个项目的 Claude 窗口会创建一个锁文件（存储 PID 和脚本路径）
2. 点击按钮时，先检查锁文件是否存在
3. 如果存在，通过 PID 找到窗口并激活
4. 如果不存在或激活失败，则创建新窗口
5. 窗口关闭时自动清理锁文件
